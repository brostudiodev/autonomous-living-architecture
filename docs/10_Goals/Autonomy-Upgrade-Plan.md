# Autonomy Upgrade Plan

## Purpose

Strategic roadmap for increasing system autonomy levels across all domains (Finance, Household, Schedule, Health).

## Autonomy Levels

| Level | Name | Description | Approval Required |
|-------|------|-------------|------------------|
| 1 | Manual | User does everything | N/A |
| 2 | Assisted | System suggests, user decides | Always |
| 3 | Semi-Autonomous | System acts, user approves | Threshold-based |
| 4 | Autonomous | System acts automatically | Safety guardrails only |
| 5 | Predictive | System anticipates and prevents | Minimal oversight |

## Current State (Q1 2026)

### Finance (G05)
- **Level 3:** Budget alerts → Telegram approval → Action
- **Level 4:** Auto-categorization (95% accuracy)
- **Level 5:** Autonomous Liquidity Rebalancing (NEW - Mar 27)

### Household (G03)
- **Level 3:** Low stock → Telegram approval → Purchase
- **Level 4:** Predictive restocking (Active Confidence Buffering)
- **Level 5:** Autonomous Procurement for critical items (NEW - Mar 27)

### Schedule (G10)
- **Level 2:** Suggestions via Digital Twin
- **Level 3:** Focus mode automation (G08 integration)
- **Target Q4:** Level 4 adaptive scheduling

### Health (G07)
- **Level 3:** Anomaly detection → Alert
- **Level 4:** Readiness-based training recommendations
- **Target Q4:** Level 4 autonomous health optimization

## Upgrade Criteria

For each domain, upgrade criteria include:
1. **Accuracy:** >95% decision accuracy over 30 days
2. **Safety:** Zero critical failures in test period
3. **Trust:** User approval rate >90%
4. **Monitoring:** Full observability into decisions
