# SOP: Daily Dashboard Review (G11)

## Overview
This SOP outlines the morning routine for reviewing the Digital Twin dashboard in Obsidian. The review ensures the user is aligned with the system's "Morning Mission" and biological reality.

## Goal
To achieve zero "Decision Fatigue" by following the system's prioritized recommendations for the first 2-4 hours of the day.

## Dashboard Components (Obsidian Daily Note)

### 1. 📡 Connectivity Status (%%CONN%%)
- Verify `Digital Twin: ✅ Online`.
- Check if `Scripts: ✅ Fresh` or if any issues require a manual `/sync`.

### 2. 🎯 Morning Mission (%%MISSION%%)
- Read the high-level directive (e.g., "Complete G05 documentation audit").
- Review the `RICH_MISSION` context for deep links and data support.

### 3. 🧠 Biological Reality (%%REFLECTION%%)
- Check Sleep Score and HRV ms.
- Adjust intensity of the day if `MVD Mode` (Minimum Viable Day) is active.

### 4. ⚡ Quick Wins (%%QUICK_WINS%%)
- Execute the top 3 items in the "Execution Zone" (Logistics, Pantry, or Roadmap tasks).
- Use the `One-Click Action` buttons if applicable.

### 5. 📥 System Approvals (%%PENDING%%)
- Resolve any pending human approvals for autonomous rebalances or cart population.

## Step-by-Step Procedure
1. **Open Obsidian:** Navigate to the current daily note (Alt+D).
2. **Review Metrics:** Glance at biometrics to gauge energy levels.
3. **Execute Quick Wins:** Complete the low-friction tasks listed in the %%QUICK_WINS%% block.
4. **Accept Suggestions:** Click the relevant `shell-command:` buttons to execute system-proposed actions (e.g., `Populate Shopping Cart`).
5. **Start Deep Work:** Transition to the `Morning Mission` for high-cognitive output.

## Frequency
- **Daily:** Every morning between 06:00 AM and 08:30 AM.
