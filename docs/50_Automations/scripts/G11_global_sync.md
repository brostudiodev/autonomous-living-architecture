# G11: Global System Synchronization

## Purpose
Acts as the primary orchestrator for the entire autonomous ecosystem. It ensures all data ingestion, analysis, and enforcement scripts run in the correct sequence to maintain a fresh Digital Twin state.

## Triggers
- **When:** Scheduled via Crontab (typically via `G11_obsidian_safe_sync.py`):
    - **Daily:** 06:00, 13:00, 16:00.
- **Manual:** `python3 G11_global_sync.py`
- **Wrapper:** `python3 G11_obsidian_safe_sync.py` (Preferred for file safety).
- **API:** Triggered via `/sync` endpoint in `G04_digital_twin_api`.

## Inputs
- **Sync Registry:** A hardcoded dictionary of scripts and their execution flags.
- **System Time:** Used to determine if Sunday-specific tasks (Weekly Review) should run.
- **Environment:** Requires the correct Python virtual environment.

## Processing Logic
1. **Health Audit:** Triggers `G11_self_healing_supervisor.run_audit()` to verify system integrity before syncing.
2. **Auto-Documentation:** Scans for undocumented scripts via `G12_auto_documenter`.
3. **Retry-Aware Execution Loop (3 Attempts):**
    - The script enters a loop to ensure data freshness (specifically biometrics).
    - **Step A:** Executes all registered data sync scripts (Finance, Health, Pantry, Logistics, etc.).
    - **Step B:** Executes enforcement and advisory scripts (e.g., `G08_environment_advisor.py`, `G03_appliance_monitor.py`, `G10_focus_enforcer.py`).
    - **Step C:** Calls `check_data_freshness()` via the Digital Twin Engine.
    - **Step D:** If data is fresh, it breaks the loop. If stale, it **waits 10 minutes** and retries Attempt #2 (and #3 if needed).
4. **Finalization (Daily Note):**
    - Only after the sync loop completes (either through success or reaching max attempts), it executes `autonomous_daily_manager.py`.
    - This ensures the Daily Note always has the absolute latest data available at that moment.
5. **Logging:** Records success/failure of each script in the `system_activity_log` table.
6. **Report Generation:** Aggregates results into a Markdown report for API/CLI consumption.

## Outputs
- **Activity Logs:** Entries in `digital_twin_michal.system_activity_log`.
- **Markdown Report:** Summary of all sync operations and durations.
- **Data Freshness:** Updates `updated_at` timestamps across all domain databases.
- **New Decisions:** Proposals generated by `G11_decision_proposer.py` (integrated in sync).

## Error Handling
| Failure Scenario | Detection | Response | Alert |
|---|---|---|---|
| Script Timeout | Subprocess timeout (120s) | Terminate script, log failure | Report entry |
| Global Crash | Exception in main loop | Log to file, exit | API `/status` check |
| DB Offline | Connection error | Skip DB-dependent scripts | Health Audit |

## Monitoring
- **Success metric:** All registered scripts show "✅ Success" in the daily sync report.
- **Drift detection:** `G04_digital_twin_engine` checks for stale data (>1h).

## Manual Fallback
Each sub-script can be run manually from the `scripts/` directory if the global orchestrator fails.
