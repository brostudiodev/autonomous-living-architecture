# 💸 G05 Pre-emptive Financial Rebalancer

## 🎯 Purpose
Automatically reallocates budget from low-priority surpluses to high-priority categories facing predicted breaches. This shifts the financial system from "Post-Mortem Analysis" to "Active Treasury Management."

## 🏗️ Architecture
- **Trigger:** Triggered after `G05_budget_friction_predictor.py` completes.
- **Logic:** Analyzes `v_budget_performance` for `predicted_utilization_pct > 100`.
- **Action:** Executes SQL `UPDATE` to shift `budget_amount` between categories in the current month.

## 🛠️ Implementation Details
- **Script:** `scripts/G05_preemptive_rebalancer.py`
- **Language:** Python 3
- **Primary Tool:** `psql` (complex SQL views)

## 🔗 Dependencies
- **Database:** `autonomous_finance`
- **View:** `v_budget_performance` (must be up-to-date)
- **Notification:** `G04_digital_twin_notifier.py`

## ⚠️ Failure Modes & Recovery
- **Insufficient Surplus:** Script notifies user that a breach is predicted but no surplus was found to cover it.
- **DB Conflict:** Uses `ON CONFLICT` or specific `UPDATE` logic to prevent budget duplication.

## 🔄 Rollback & Maintenance
- **Rollback:** Manual budget adjustment in the `budgets` table or Google Sheets (synced back via G05_finance_sync).
- **Maintenance:** Priority levels for categories are defined in the SQL view logic.
