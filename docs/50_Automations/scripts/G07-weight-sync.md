# G07: Weight Sync (Withings)

## Purpose
Automates the retrieval of weight and body fat percentage from the Withings Health API and syncs it directly to the PostgreSQL `autonomous_health` database.

## Triggers
- Scheduled: Part of the morning briefing/daily fill workflow.
- Manual: `python3 scripts/G07_weight_sync.py`

## Inputs
- Withings Health API (via OAuth tokens in `withings_tokens.json`)
- PostgreSQL database: `autonomous_health`

## Processing Logic
1. Authenticates using stored OAuth credentials from `withings_to_sheets.py`.
2. Fetches the latest measurement groups from Withings.
3. Extracts `WEIGHT` and `FAT_RATIO` measures.
4. Performs an `UPSERT` into the `biometrics` table in `autonomous_health`, keyed by `measurement_date`.

## Outputs
- Data entries in `biometrics` table (weight_kg, fat_percentage).
- Console log confirming sync count.

## Dependencies
- [withings-api](https://pypi.org/project/withings-api/) (Python library)
- [S03 Data Layer](../../20_Systems/S03_Data-Layer/README.md)
- Credentials: `withings_tokens.json` (generated by `withings_to_sheets.py`)

## Failure Modes
| Failure Scenario | Detection | Response | Alert |
|---|---|---|---|
| Token Expired | Auth error in logs | Run `withings_to_sheets.py` manually to re-auth | Log Error |
| DB Unavailable | Connection timeout | Check PostgreSQL container status | Log Error |
| No New Data | 0 rows synced | System assumes no new weigh-in occurred | Log Info |
