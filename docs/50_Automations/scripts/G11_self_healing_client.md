# 🤖 Automation Spec: G11_self_healing_client.py

## 📝 Overview
**Purpose:** Dispatches system failure reports to an external n8n orchestrator for autonomous error analysis and recovery.
**Goal Alignment:** G11 Meta-System Integration (Self-Healing), G04 Digital Twin (System Stability).

## ⚡ Technical Details
- **Language:** Python
- **Triggers:** Triggered by `fill-daily.sh` when a health audit or manager script fails.
- **Databases:** None (reads from local `_meta/automation_health.json`).
- **Dependencies:** `requests`, `json`, `dotenv`.
- **Target:** `N8N_WEBHOOK_URL` defined in `.env`.

## 🛠️ Logic Flow
1. **Load Audit:** Reads the latest `automation_health.json` report generated by `G11_self_healing_supervisor.py`.
2. **Check Status:** If the system is `Healthy`, the script exits.
3. **Dispatch:** If `Issues Detected`, it packages the entire failure list (script name, error message, stack trace) into a JSON payload.
4. **Trigger n8n:** Sends a POST request to the n8n webhook.
5. **Log:** Records the dispatch success/failure in the system activity log.

## 📤 Outputs
- **Webhook Payload:** Sent to n8n for LLM processing.
- **Activity Log:** Entry in `system_activity_log`.

## ⚠️ Known Issues / Maintenance
- **Connectivity:** Requires the n8n instance to be reachable from the host.
- **Retry Logic:** Currently relies on the once-daily execution of `fill-daily.sh`.
